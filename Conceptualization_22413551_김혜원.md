## 1. Business purpose
### 1.1 Project background
 바쁜 현대인들의 일상에서 식사는 가장 기본적이고 반복적인 행위이지만 역설적이게도 많은 사람들이 하루 중 가장 자주 고민하는 문제 중 하나가 바로 "오늘 뭐 먹지?"라는 질문이다. 음식 선택은 단순히 배를 채우는 행위를 넘어, 개인의 기분, 건강 상태, 경제적 여건, 일정, 날씨, 동행 여부 등 다양한 요소의 영향을 받는다. 배달 어플리케이션, 맛집 추천 플랫폼, SNS 기반 리뷰 서비스가 보편화된 현재는 음식점과 메뉴에 대한 정보가 과도할 정도로 넘쳐나는 시대이고 과거에는 선택지가 부족해 고민이 적었다면 현재는 지나치게 많은 선택지와 정보 때문에 오히려 결정을 내리기 어려운 상황이 빈번하게 발생한다. 사용자는 음식 종류를 정하기도 전에 한식, 중식, 양식, 분식, 패스트푸드, 디저트 등 수많은 범주 사이에서 망설이다 지치게 되고 또한 예산 제한, 현재 위치, 알레르기 여부, 다이어트 여부, 호불호 식재료, 혼밥 혹은 단체 식사와 같은 조건이 추가되면 단순히 메뉴 선택하는 행위가 상당한 인지적 부담을 유발하게 된다. 
 
 필자의 경우에는 어머니가 채식주의자라 고기가 들어가지 않은 음식이 매우 한정적이라 여행을 가서도 메뉴 선택에만 시간을 많이 뺏기곤 했다. 이러한 문제는 단순히 '추천 목록'을 제공하는 기존 방식만으로는 충분히 해결되기 어렵다. 대부분의 기존 메뉴 추천 서비스는 랜덤으로 돌려서 나오는 것을 추천하거나, 대략적인 카테고리를 고르고 동시에 9가지 정도의 메뉴를 추천한 뒤에 구미가 당길만한 메뉴가 나올 때까지 새로고침하는 방식이었다. 이는 사용자가 지금 어떤 감정 상태인지, 어떤 식사 목적을 가지고 있는지에 대한 맥락적 이해는 부족한 편이다. 

 가령 사용자가 "오늘 날씨가 춥고, 몸이 좀 피곤해서 따뜻한 국물류 음식이 먹고 싶다."거나, "점심시간이 짧아서 간단하지만 든든한 메뉴가 필요하다.", "다이어트 중인데 너무 맛없는 건 싫다.", "어제 까르보나라 파스타를 먹어서 느끼한 건 먹기 싫다." 등등 이와 같이 자연스러운 방식으로 자신의 상황을 설명하더라도 기존 시스템은 이를 반영하여 메뉴를 추천하지 못하기에 결국 사용자는 서비스를 이용하고도 다시 여러 후보를 스스로 비교하고 판단해야 해서 실질적인 편의성을 증가시켰다고 보기는 어려웠다. 이 프로젝트는 바로 이러한 현실적인 불편함에서 출발한다.


### 1.2 Goal
 본 서비스의 주요 개발 목적은 사용자가 매일 반복적으로 겪는 메뉴 선택의 부담을 줄이고, 개인의 상황과 취향을 반영한 맞춤형 추천을 통해 보다 효율적이고 만족도 높은 식사 결정을 지원하는 데 있다. 단순히 메뉴 목록 나열 수준을 넘어, 사용자가 AI와의 자연스러운 대화를 통해 요구사항을 전달하면 이를 분석하여 최적의 선택지를 제안함으로써 일상 속 작은 불편함을 해결하고자 한다.


### 1.3 Target Market
 타겟층은 반복적으로 메뉴 선택에 어려움을 겪는 일반 사용자 전반이다. 특히 바쁜 일정 속에서 빠르게 식사를 결정해야 하거나, 선택지가 너무 많아 오히려 결정을 내리기 어려워하는 사람들을 핵심 사용자층으로 설정할 수 있다. 부가적으로 건강 관리에 관심이 있는 사용자, 특정 식단 제한이 있는 사용자(당뇨질환자, 채식주의자 등), 다이어터와 같은 사용자도 포함한다.

## 2. System context diagram
사용자는 시스템에 현재 먹고 싶은 메뉴 조건을 입력한다. 시스템은 입력 내용을 분석하여 메뉴 데이터와 비교하고 조건에 맞는 메뉴 목록을 추출한 뒤 우선순위를 반영하여 추천 결과를 반환한다. 사용자가 추천 결과에 대해 다시 조건을 추가하거나 재추천을 요청하면 시스템은 동일한 흐름을 반복 수행한다.

![Diagram](https://github.com/user-attachments/assets/47c894a0-1333-4939-859d-dcb34b8ccb34)


| 용어                                      | 내용                                                                    |
| ----------------------------------------- | -------------------------------------------------------------------------------- |
| User                | 사용자가 시스템에 메뉴 추천을 요청하거나 조건을 입력하는 주체로, 로그인, 조건 입력, 추천 확인, 재추천 요청 등의 상호작용을 수행.    |
| Login          | 사용자가 자신의 계정 정보를 입력하여 시스템에 인증을 요청하는 과정으로, 시스템은 입력된 정보를 검증하여 로그인 성공 여부를 판단.      |
| Register        | 사용자가 새로운 계정을 생성하기 위해 ID와 비밀번호 등의 정보를 시스템에 전달하는 과정이다.      |
| InputCondition         | 사용자가 원하는 음식 조건(선호 음식, 기피 음식, 식단 제한 등)을 자연어 또는 키워드 형태로 시스템에 전달하는 입력 과정.         |
| AnalyzeCondition      | 시스템이 사용자 입력을 분석하여 의미 있는 키워드와 조건(맛, 카테고리, 상황 등)을 추출하고, 이를 구조화된 데이터 형태로 변환하는 과정. |
| SavePreference      | 시스템이 사용자 입력 또는 선택 결과를 기반으로 선호 및 비선호 정보를 저장하여 이후 추천 과정에 활용하는 기능.                |
| ManageMenu         | 시스템이 메뉴 데이터베이스에 저장된 메뉴 정보를 추가, 수정, 삭제하며 관리하는 기능으로, 추천 시스템의 기반 데이터를 유지.         |
| FilterMenu         | 시스템이 사용자 조건에 맞지 않는 메뉴를 제거하고, 조건을 만족하는 메뉴만을 후보군으로 추출하는 과정.                      |
| RecommendMenu | 필터링된 메뉴 후보군을 기반으로 조건 일치도 및 선호도를 고려하여 최종 추천 메뉴 리스트를 생성하는 핵심 기능.                 |
| ViewRecommendation  | 시스템이 생성한 추천 결과를 사용자에게 시각적으로 제공하는 과정으로, 사용자는 이를 확인하고 선택 가능.                    |
| ReRecommend    | 사용자가 기존 추천 결과에 만족하지 않을 경우 새로운 조건을 반영하거나 기존 결과를 제외하여 다시 추천을 요청하는 과정.            |
| System                | 사용자 입력을 처리하고, 조건 분석, 메뉴 필터링, 추천 생성, 결과 출력 등 전체 추천 프로세스를 수행하는 핵심 주체.            |

## 4. Concept of operation
### 1) Register
| 항목       | 내용                                              |
| -------- | ----------------------------------------------- |
| Purpose  | 새로운 사용자가 시스템을 사용할 수 있도록 계정을 생성하기 위함             |
| Approach | 사용자가 ID와 비밀번호를 입력하면 시스템이 이를 저장하여 사용자 데이터를 생성한다. |
| Dynamics | 사용자가 처음 시스템을 사용할 때 동작                           |
| Goals    | 로그인 가능한 사용자 계정을 생성한다.                           |

### 2) Login
| 항목       | 내용                                      |
| -------- | --------------------------------------- |
| Purpose  | 등록된 사용자인지 확인하고 개인화 기능을 사용할 수 있도록 하기 위함  |
| Approach | 사용자가 입력한 계정 정보를 사용자 데이터와 비교하여 인증을 수행한다. |
| Dynamics | 사용자가 시스템 접근 시 동작                        |
| Goals    | 사용자 인증을 완료하고 시스템 사용 권한을 부여한다.           |


### 3) Input User Condition
| 항목       | 내용                             |
| -------- | ------------------------------ |
| Purpose  | 사용자의 현재 요구사항을 입력받기 위함          |
| Approach | 사용자가 텍스트 또는 선택형 방식으로 조건을 입력한다. |
| Dynamics | 프로그램 실행 후 추천 요청 시 동작           |
| Goals    | 추천에 필요한 핵심 조건을 수집한다.           |

### 4) Analyze Condition
| 항목       | 내용                                            |
| -------- | --------------------------------------------- |
| Purpose  | 사용자 입력에서 의미 있는 조건을 추출하기 위함                    |
| Approach | 키워드 매칭 또는 간단한 자연어 처리 방식으로 선호/비선호/상황 조건을 분류한다. |
| Dynamics | 입력 직후 자동 실행                                   |
| Goals    | 추천 로직에 전달할 구조화된 조건 정보를 생성한다.                  |

### 5) Filter Menu
| 항목       | 내용                                          |
| -------- | ------------------------------------------- |
| Purpose  | 전체 메뉴 중 사용자 조건에 맞지 않는 메뉴를 제거하기 위함           |
| Approach | 카테고리, 식재료, 식단 제한, 상황 태그를 기준으로 메뉴 후보를 필터링한다. |
| Dynamics | 조건 분석 이후 수행                                 |
| Goals    | 추천 정확도를 높일 수 있는 후보 메뉴군을 생성한다.               |


### 6) Recommend Menu
| 항목       | 내용                                              |
| -------- | ----------------------------------------------- |
| Purpose  | 사용자에게 최종 메뉴 추천 결과를 제공하기 위함                      |
| Approach | 필터링된 메뉴 중 조건 충족 수, 우선순위 점수 등을 바탕으로 상위 메뉴를 제시한다. |
| Dynamics | 메뉴 필터링 완료 후 수행                                  |
| Goals    | 사용자가 바로 선택 가능한 수준의 추천 결과를 제공한다.                 |

### 7) Request Re-recommendation
| 항목       | 내용                                        |
| -------- | ----------------------------------------- |
| Purpose  | 첫 번째 추천 결과가 만족스럽지 않을 경우 대안을 제시하기 위함       |
| Approach | 기존 추천 결과를 제외하거나 추가 조건을 반영하여 새로운 추천을 수행한다. |
| Dynamics | 사용자가 재추천을 요청할 때 수행                        |
| Goals    | 사용자 만족도를 높이고 반복 선택 피로를 줄인다.               |

### 8) Save Preference
| 항목       | 내용                                 |
| -------- | ---------------------------------- |
| Purpose  | 반복 사용 시 개인화 추천의 기반을 마련하기 위함        |
| Approach | 사용자가 선호/비선호한 메뉴나 조건을 내부 데이터에 저장한다. |
| Dynamics | 추천 이후 피드백 입력 시 수행                  |
| Goals    | 이후 추천 결과의 적합도를 향상시킨다.              |

## 5. Problem statement
1.  **사용자 입력 해석의 불확실성 문제**: 사용자의 자연어 입력을 기반으로 추천을 수행하므로, “느끼하지 않은 음식”, “가볍지만 든든한 메뉴”와 같이 주관적이고 모호한 표현을 정확히 해석하기 어렵다. 이러한 표현은 사용자마다 의미가 다르게 해석될 수 있으며, 키워드 기반 분석 방식에서는 의도를 완전히 반영하지 못할 가능성이 존재한다. 이는 추천 결과의 정확도 저하로 이어질 수 있다.

2. **데이터 품질 및 다양성 한계**: 실제 환경에 비해 메뉴의 다양성과 데이터의 풍부함이 부족하다. 이로 인해 특정 조건에서 추천 가능한 메뉴 수가 제한되거나, 반복적인 추천 결과가 나타나는 문제가 발생할 수 있으며, 결과적으로 사용자 만족도가 저하될 수 있다.

3. **추천 모델의 정밀도 제약**: 무료로 제공되는 AI API는 호출 횟수, 응답 범위, 처리 능력 등에 제한이 존재하므로, 다양한 사용자 조건을 정밀하게 반영하는 데 제약이 있다.

4. **확장성 문제**: 현재 시스템은 소규모 데이터와 단일 사용자 환경을 기준으로 설계되었기 때문에, 사용자 수 증가 또는 데이터 규모 확장 시 성능 저하가 발생할 수 있다. 특히 추천 처리 속도와 데이터 관리 측면에서 확장성을 고려한 구조 설계가 필요하다.

5. **외부 환경 반영 한계**: 날씨, 위치, 음식점 정보와 같은 외부 API를 활용하지 않기 때문에, 실제 사용자 상황을 완전히 반영한 추천을 제공하기 어려워 항상 사용자에게 받아서 추천해야 한다. 

| 항목              | 설명                                                                 |
| --------------- | ------------------------------------------------------------------ |
| Usability       | 사용자는 복잡한 설정 없이 자연어 또는 간단한 입력만으로 추천을 받을 수 있어야 하며, 인터페이스는 직관적이어야 한다. |
| Performance     | 추천 결과는 사용자의 입력 이후 짧은 시간 내에 제공되어야 하며, 불필요한 지연 없이 빠르게 동작해야 한다.       |
| Maintainability | 메뉴 데이터 및 추천 규칙은 코드 수정 없이도 쉽게 추가 및 변경할 수 있도록 구조화되어야 한다.             |
| Scalability     | 현재는 소규모 데이터 기반이지만, 향후 메뉴 수 증가 및 기능 확장에 대응할 수 있는 구조를 가져야 한다.        |
| Reliability     | 동일한 조건 입력에 대해 일관된 추천 결과를 제공해야 하며, 시스템 오류 없이 안정적으로 동작해야 한다.         |
| Extensibility   | 향후 외부 API 연동, 사용자 학습 기능, 고도화된 AI 모델 적용이 가능하도록 확장성을 고려해야 한다.        |

## 6. Glossary
| 용어                                | 설명                                                                          |
| --------------------------------- | --------------------------------------------------------------------------- |
| Recommendation System             | 사용자의 요구사항, 선호도, 조건 등을 분석하여 적절한 메뉴를 선택하고 제안하는 시스템을 의미한다.                     |
| Natural Language Input            | 사용자가 일상 언어 형태로 입력하는 문장 데이터로, 본 시스템에서는 이를 분석하여 조건 정보로 변환한다.                | 
| Menu Database                     | 메뉴 이름, 카테고리, 재료, 태그 등의 정보를 저장하는 데이터 집합으로, 추천 과정의 기반이 된다.                    |
| Filter                            | 사용자 조건에 맞지 않는 메뉴를 제거하여 후보군을 줄이는 과정이다.                                       |
| Preference                        | 사용자가 선호하는 음식 또는 조건으로, 추천 결과에 긍정적으로 반영되는 요소이다.                               |
| Non-preference                    | 사용자가 기피하거나 제한하는 음식 또는 조건으로, 추천 과정에서 제외되는 요소이다.                              |
| Keyword Matching                  | 사용자 입력에서 특정 단어를 기준으로 조건을 판별하는 방식으로, 자연어 처리의 단순화된 형태이다.                      |
| Rule-based System                 | 사전에 정의된 규칙을 기반으로 입력을 처리하고 결과를 도출하는 시스템 구조이다.                                |
| Re-recommendation                 | 사용자가 기존 추천 결과에 만족하지 않을 경우 새로운 조건을 반영하여 다시 추천을 수행하는 기능이다.                |   
| Context                           | 사용자의 현재 상황(시간, 기분, 날씨, 식사 목적 등)을 의미하며, 추천 결과에 영향을 미치는 요소이다.                 |
