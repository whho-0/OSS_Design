# 1. Introduction <a name="intro"></a>
매일 식사 때마다 "오늘 뭐 먹지?"라는 사소하지만 반복적인 고민을 누구나 겪는다. 메뉴 선택은
개인의 건강 상태, 알레르기·식이 제한, 그날의 기분 및 선호하는 음식 종류등 다양한 변수에 의해
좌우되지만, 기존의 단순 랜덤 메뉴 추천기들은 이러한 개인적 맥락을 전혀 반영하지 못한다.

본 시스템 AI 메뉴 추천봇은 사용자의 건강 정보와 식이 제한을 회원가입 단계에서 한 번
등록해 두고, 추천을 요청할 때마다 제외할 카테고리, 국물 유무, 면/밥 선호, 음식 온도 선호를 단계별 UI로 입력받아, 이를 종합한 프롬프트를 LLM에 전달하여 개인화된 메뉴 3가지를 추천한다. 또한 추천 결과가 마음에 들지 않을 경우, 이전 추천 맥락(대화 히스토리)을 유지한 채 후속 채팅으로 메뉴를 정제(refine)할 수 있다.

# 2. Class Diagram <a name="cd"></a>

<img width="8192" height="4100" alt="2_class_diagram" src="https://github.com/user-attachments/assets/6fbffe50-d814-4861-bab0-ea9909dd69cc" />

### 1) DatabaseManager
SQLite 데이터베이스 연결과 스키마 초기화를 담당하는 데이터 접근 계층(DAO) 클래스이다.

- **Attributes**
  - `db_path:str` : 데이터베이스 파일 경로 (기본값 `menu_bot.db`)
- **Methods**
  - `conn():Connection` : `row_factory`를 설정한 SQLite 커넥션을 반환한다.
  - `_init_db():void` : `users`, `user_preferences`, `chat_sessions`, `messages` 4개 테이블을 생성한다.

---

### 2) User
회원 정보(`users` 테이블)에 대한 등록·조회 및 비밀번호 해싱을 담당하는 모델 클래스이다.

- **Methods**
  + `_hash(pw):str` : SHA-256으로 비밀번호를 해싱한다.
  + `register(username, password):(bool, str)` : 신규 회원을 삽입한다. 중복 시 `IntegrityError`를 잡아 실패를 반환한다.
  + `login(username, password):(bool, dict)` : ID/해시 일치 여부를 확인하고 사용자 정보를 반환한다.

---

### 3) AuthManager
입력 유효성 검증을 추가한 인증 비즈니스 로직 클래스로, `User`를 위임(delegate)한다.

- **Methods**
  + `register(username, password, confirm):(bool, str)` : ID 2자 이상, 비밀번호 4자 이상, 비밀번호 일치를 검증 후 `User.register` 호출.
  + `login(username, password):(bool, dict)` : 빈 입력 검증 후 `User.login` 호출.

---

### 4) UserPreference
사용자의 건강·식이 정보(`user_preferences` 테이블)와 초기 설정 완료 여부를 관리한다.

- **Methods**
  + `get(user_id):dict` : 사용자 선호 레코드를 반환한다.
  + `is_setup_done(user_id):bool` : 최초 건강정보 설정 완료(`is_setup_done`) 여부를 반환한다.
  + `save_health(user_id, health_info):void` : UPSERT로 건강정보를 저장하고 설정 완료 플래그를 1로 설정한다.

---

### 5) MenuFilter
사용자의 선택 조건을 LLM에 전달할 프롬프트 문자열로 조립하는 유틸리티 클래스이다.

- **Attributes**
  + `CATEGORIES:list` : 한식·중식·일식·양식·분식·아시안·패스트푸드
- **Methods**
  + `build_prompt(disliked, soup, type, temp, health):str` : 제외 카테고리, 국물·면밥·온도 선호, 건강 제한을 ` | ` 구분자로 결합한다.

---

### 6) ConversationHistory
대화 메시지 목록을 보관하는 보조 모델 클래스이다. (`role`/`content` 딕셔너리 리스트)

- **Methods**
  + `add(role, content):void` : 메시지를 추가한다.
  + `to_list():list` : 메시지 목록의 복사본을 반환한다.

---

### 7) GroqClientWrapper
Groq LLM API 호출을 캡슐화하는 외부 연동 클래스이다.

- **Attributes**
  + `client:Groq`, `model:str = "llama-3.3-70b-versatile"`
- **Methods**
  + `chat(messages):str` : `temperature=0.8`, `max_tokens=1024`로 채팅 완성을 요청하고 응답 텍스트를 반환한다.

---

### 8) MenuRecommender
메뉴 추천과 후속 정제를 담당하는 핵심 비즈니스 로직 클래스이다.

- **Attributes**
  + `SYSTEM:str` : JSON 형식 강제 시스템 프롬프트
- **Methods**
  + `recommend(filter_prompt):dict` : 조건 프롬프트로 메뉴 3종 추천을 요청하고, 응답에서 JSON을 파싱한다. 실패 시 fallback 결과를 반환한다.
  + `chat_refine(history, user_msg, health):str` : 건강정보와 직전 20개 히스토리를 주입하여 대화형 정제 응답을 생성한다.

---

### 9) ChatSession
추천 세션(`chat_sessions`)과 메시지(`messages`)의 CRUD를 담당한다.

- **Methods**
  + `create(user_id, title, filter_data):int` : 새 세션을 생성하고 `lastrowid`를 반환한다.
  + `get_sessions(user_id):list` : 최근 30개 세션을 반환한다.
  + `add_message(session_id, role, content):void` : 메시지를 저장한다.
  + `get_messages(session_id):list` : 세션의 전체 메시지를 시간순으로 반환한다.

---

### 10) AppController
모든 하위 컴포넌트를 생성·조립(Composition)하는 Facade 클래스이다. 전역 `ctrl` 인스턴스로
사용되며, Flask 라우트는 이 단일 진입점을 통해서만 도메인 로직에 접근한다.

# 4. State Machine Diagram <a name="smd"></a>

<img width="4903" height="4957" alt="3_회원가입 (2)" src="https://github.com/user-attachments/assets/5aec0327-6e9b-46f9-a531-55557e752931" />

앱을 실행하면 `/api/me`로 세션을 확인하여 **Auth** 화면에서 시작한다. 신규 회원은 건강정보

마친 기존 회원은 로그인 즉시 **Main**으로 진입한다.

# 5. Implementation Requirements <a name="ir"></a>

## 1) S/W platform requirements
- **OS** : Windows 10+ / macOS / Linux (크로스 플랫폼)
- **Language** : Python 3.8 이상
- **Web Framework** : Flask
- **Database** : SQLite 3 (내장)
- **LLM API** : Groq API — 모델 `llama-3.3-70b-versatile`
- **주요 라이브러리** : flask, groq, hashlib, sqlite3, json (표준 라이브러리 포함)
- **Client** : 모던 웹 브라우저 (Chrome, Edge, Safari 등 — fetch/ES6 지원)

## 2) H/W platform requirements
- **CPU** : Intel i3 이상 (또는 동급)
- **RAM** : 4GB 이상
- **HDD / SSD** : 1GB 이상 여유 공간
- **Network** : Groq API 호출을 위한 인터넷 연결 필요

# 6. Glossary <a name="glossary"></a>

| Term | Description |
| --- | --- |
| Flask | Python 기반 경량 웹 프레임워크. 라우팅과 세션, 템플릿 렌더링을 제공한다. |
| LLM | Large Language Model. 본 시스템은 Groq의 LLaMA 3.3 70B 모델을 사용한다. |
| Facade | 복잡한 하위 시스템을 단일 진입점으로 감싸는 디자인 패턴. `AppController`가 이에 해당한다. |
| UPSERT | 레코드가 존재하면 UPDATE, 없으면 INSERT 하는 갱신 전략. `save_health`에서 사용한다. |
| SHA-256 | 256비트 단방향 해시 함수. 비밀번호 저장에 사용한다. |
| Session | 서버가 클라이언트별 로그인 상태를 유지하는 메커니즘. Flask `session` 사용. |
| Prompt | LLM에 전달하는 입력 텍스트. `MenuFilter`가 조건을 조립하여 생성한다. |
| Fallback | LLM 응답 JSON 파싱 실패 시 기본 오류 결과를 반환하는 예외 처리. |
